# 米游社游戏状态功能

应用内置三个游戏状态按键功能：`原神状态`、`星铁状态`、`绝区零状态`。它们在侧栏同一个 `游戏` 卡片中，但共享同一份米游社登录状态。

## 登录

- 在任意游戏功能的 `功能参数` 区点击 `生成登录二维码`。
- 使用米游社 App 扫码并确认后，应用会把登录会话保存到系统 Keychain。
- 重新登录会清除当前会话并生成新的二维码。
- 如果查询接口返回登录失效，应用会清除 Keychain 中的会话，并让所有游戏按键进入需要重新登录的状态。

## 查询内容

| 游戏 | 体力字段 | 日常字段 |
| --- | --- | --- |
| 原神 | 树脂 | 每日委托 |
| 崩坏：星穹铁道 | 开拓力 | 每日实训 |
| 绝区零 | 电量 | 活跃度 |

绝区零的日常完成状态使用 `vitality.current/max`，不是 `bounty_commission`。体力、开拓力和电量都允许超过上限，显示层不能把当前值裁剪到最大值。

## 接口来源

Swift 客户端优先使用实时便笺接口；如果实时便笺失败，会尝试对应游戏的小组件接口。小组件返回的绝区零电量可能受接口来源限制，配置区会标出 `该接口可能返回受限体力值`。

## 工程边界

- `MihoyoGameClient` 负责扫码登录、绑定角色、状态查询和字段映射。
- `H200ConnectionModel` 负责共享登录状态、二维码轮询、登录失效处理和按键刷新。
- 单个按键配置只保存功能类型；最近一次游戏查询结果只用于运行时显示，不写入按键配置持久化。
