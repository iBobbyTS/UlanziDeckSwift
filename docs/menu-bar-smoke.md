# 菜单栏生命周期 smoke

涉及 `LSUIElement`、`MenuBarExtra`、主窗口 scene 或 `UlanziDeckAppState` 所有权的改动，在发布前按以下顺序手工验证：

1. 启动应用，确认 Dock 中没有常驻图标，菜单栏出现一个 `Ulanzi Deck` 图标，主窗口可见。
2. 记录当前 H200 连接状态或修改一个无风险的界面配置，然后关闭主窗口。
3. 等待至少 3 秒，确认应用进程仍在运行，H200 没有因窗口关闭而恢复默认状态。
4. 从菜单栏选择“打开主窗口”，确认只出现一个主窗口，并且关闭前的连接状态与配置仍然存在。
5. 连续点击“打开主窗口”两次，确认不会产生第二个主窗口。
6. 再次启动同一 app bundle，确认第二个进程显示重复实例提示，且不会建立第二条 H200 连接。
7. 从菜单栏选择“退出 Ulanzi Deck”，确认进程退出、菜单栏图标消失，随后可正常重新启动。

真实 H200 不在场时，第 3 步至少确认窗口关闭后菜单栏入口仍可用；有设备时必须同时验证连接与 keep-alive。
