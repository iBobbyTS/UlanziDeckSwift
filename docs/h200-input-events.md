# H200 按键事件回传

应用启动同步成功后，会继续持有 H200 协议 HID 接口，并注册输入报告回调。只有 H200 物理按键事件会触发按键功能；app 内格子点击只切换 UI 选中边框和功能参数面板目标，不触发按键功能。app 内格子右击选择“删除”，或在右侧功能列表再次点击已选中的功能项，都会把该按键清为空功能并重新同步设备显示。

## 输入包格式

当前只解析参考仓库确认的 `0x0101` 按键输入包：

```text
[0x7c 0x7c][cmd:0x0101 u16 BE][length u32 LE][state u8][index u8][type u8][action u8]
```

字段约定：

- `index`：LCD 格子从 `0` 开始编号，当前 H200 原型映射到应用内按键 `1...14`。
- `type == 0x02`：旋钮事件；其他值按普通按键处理。
- `action == 0x01`：按下；`0x02` 和 `0x03` 分别表示旋钮左转和右转；其他值按释放处理。

## 当前行为

- `index 0...13` 映射到应用内 `1...14`。
- `press` 进入按下态并启动 1 秒长按计时。
- `release` 结束按下态；如果没有触发长按，则 tally 当前值加 `1`。
- 长按满 1 秒会把 tally 当前值重置为该按键配置的默认数值；随后释放不会再加 `1`。
- 无功能按键不会进入按下态，不启动长按计时，也不会触发 tally 或额外同步。
- 旋钮、页面键和未知索引暂不触发 14 格原型 UI。
- 物理按键事件不会切换功能参数面板的当前按键；功能参数面板只跟随 app 内格子点击选择。
- app 内格子选中态通过外层边框表示，不参与 H200 图标渲染，设备端不会因为 app 选中或 deck 点击出现蓝色背景。

app 内格子的视觉预览复用发包时的 PNG 渲染逻辑，确保 app 中看到的内容和发送到 H200 的图像一致。tally 值变化后，当前原型会先保存本地配置，再重新发送完整按键包，让设备显示、app 可见状态和下次启动恢复状态保持一致；后续可改为 `0x000d` 局部更新以降低刷新成本。
