# H200 按键事件回传

应用启动同步成功后，会继续持有 H200 协议 HID 接口，并注册输入报告回调。只有 H200 物理按键事件会触发按键功能；app 内格子单击只切换 UI 选中边框和功能参数面板目标，不触发按键功能。app 内双击页面文件夹会进入子页面，双击返回键会返回上一级页面。app 内格子右击选择“删除”，或在右侧功能列表再次点击已选中的功能项，都会把可删除按键清为空功能并重新同步设备显示；返回键由子页面自动维护，不能删除或重新分配功能。

## 输入包格式

当前只解析参考仓库确认的 `0x0101` 按键输入包：

```text
[0x7c 0x7c][cmd:0x0101 u16 BE][length u32 LE][state u8][index u8][type u8][action u8]
```

字段约定：

- `index`：LCD 格子从 `0` 开始编号，当前 H200 原型映射到应用内按键 `1...14`。
- `type == 0x02`：旋钮事件；其他值按普通按键处理。
- `action == 0x01`：按下；`0x02` 和 `0x03` 是旋钮左转和右转事件，目前解析为释放或忽略，不作为主页翻页入口；其他值按释放处理。

## 当前行为

- `index 0...13` 映射到应用内 `1...14`。
- `press` 进入按下态并启动 1 秒长按计时。
- `release` 结束按下态；如果没有触发 tally 长按重置，则按当前按键功能执行短按动作。
- 计数器功能短按会让 tally 当前值加 `1`。
- 打开文件夹功能短按会用 Finder 打开已配置的文件夹；如果还没有配置文件夹，则不执行动作。
- 打开文件功能短按会用默认应用打开已配置的文件；如果还没有配置文件，则不执行动作。
- 打开网页功能短按会用默认浏览器打开已配置的网页；如果还没有配置网页地址或地址无效，则不执行动作。
- 页面文件夹功能短按会进入对应子页面，并发送该页面完整 14 格显示包；最多允许从当前主页向下嵌套 3 层。
- 返回键短按会回到父页面，并发送父页面完整 14 格显示包。返回键只存在于子页面，可在方形格之间拖拽交换，不能拖到第 14 个宽格。
- 上一页和下一页功能显示文案分别为“上一页”“下一页”，默认图标分别为 `chevron.left` 和 `chevron.right`。短按只在主页生效，会在最多 10 个主页之间切换并发送目标主页完整 14 格显示包；在功能夹内即使配置了这两个功能也不会改变页面，第一页继续上一页或最后一页继续下一页也不会越界。
- 长按满 1 秒会把 tally 当前值重置为该按键配置的默认数值；随后释放不会再加 `1`。
- 非 tally 功能不使用长按重置；按住超过 1 秒后释放仍按普通短按动作处理。
- 无功能按键不会进入按下态，不启动长按计时，也不会触发 tally 或额外同步。
- 旋钮、硬件保留页面键和未知索引暂不触发 14 格原型 UI；主页翻页只通过配置到 14 格按键上的“上一页”和“下一页”功能触发。
- 物理按键事件不会切换功能参数面板的当前按键；功能参数面板只跟随 app 内格子点击选择。
- app 内格子选中态通过外层边框表示，不参与 H200 图标渲染，设备端不会因为 app 选中或 deck 点击出现蓝色背景。

app 内格子的视觉预览复用发包时的 PNG 渲染逻辑，确保 app 中看到的内容和发送到 H200 的图像一致。tally 值、按键级显示名称/背景/高斯模糊/变暗配置、文件夹/文件/网页参数或 SMB 地址变化后，当前原型会先保存本地配置，再用 `0x000d` 局部更新包只发送当前页面变化的单个格子，让设备显示、app 可见状态和下次启动恢复状态保持一致。主页结构、功能夹结构和每页按键配置会持久化，当前停留页面不持久化；应用重启或设备重连后回到第 1 个主页。根层页面选择器显示 `1, 2, 3...` 标签，新增按钮放在页面标签右侧；右击当前主页标签可删除当前主页，删除前会弹确认。功能夹内不显示主页新增或删除入口，也不允许通过上一页/下一页功能切换主页。亮度不再是按键功能，改为顶部连接状态左侧的全局 slider。slider 变化会先立即更新 app 状态并持久化，然后在后台发送 `0x000a` 亮度包；如果发送期间 UI 又刷新到新的亮度值，只保留最新值，并在当前发送成功后补发一次，避免亮度包堆积。
